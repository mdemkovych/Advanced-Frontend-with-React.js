# Sprint 6 - Form Validation with Zod and React Hook Form

## Overview
This sprint focuses on building **robust and type-safe forms** in React using **React Hook Form** and **Zod** for validation.  
The goal is to understand how to handle form state, perform validation, and integrate schema-based validation for reliable user input handling.

---

# Learning Materials

### React Hook Form

- Introduction to **React Hook Form**
- Benefits over traditional form handling
- Setting up forms with `useForm()`
- Registering input fields
- Handling form submission
- Managing form state and errors

---

### Zod Validation

- Introduction to **Zod**
- Defining schemas for form validation
- Integrating Zod with React Hook Form
- Validating different types of input:
    - Text fields
    - Numbers
    - Emails
- Handling validation errors
- Displaying error messages in the UI

---

### Form Handling Concepts

- Controlled vs uncontrolled components
- Using `register` to bind inputs
- Accessing form values via `getValues()` and `watch()`
- Handling submission with `handleSubmit()`
- Resetting forms with `reset()`
- Conditional validation rules

---

### Fetch API Integration

- Introduction to **Fetch API**
- Sending GET and POST requests
- Handling JSON responses
- Error handling and response validation
- Using fetched data in forms
- Integrating API calls on form submission
