# Sprint 4 - JSX, Styles, SCRUM Theory

## Overview
This sprint focuses on learning the fundamentals of **JSX**, styling React components, creating the first React and Next.js applications, and understanding the basics of **SCRUM** methodology.  
The goal is to understand how JSX works in React, how to apply styles to components, how to structure a simple React/Next.js project, and how SCRUM principles are used in team development.

## Key Concepts

### JSX Basics

- JSX allows writing **UI structure inside JavaScript**
- JSX looks like HTML but follows **JavaScript rules**
- Every component must return **valid JSX**
- JSX expressions are written inside `{}`

---

### React Component Styling

- Styling can be done with:
    - external **CSS files**
    - **inline styles**
    - dynamic class assignment
- `className` is used instead of `class`
- Styles can be static or dynamic depending on component state/props
