# Sprint 8 - Tools

## Overview
This sprint focuses on essential frontend development tools used in modern JavaScript and React workflows.  
The goal is to understand what each tool is used for, how they improve development, and why they are important in real-world projects.

---

# Learning Materials

### npm

- **npm (Node Package Manager)** is used to **install, manage, and update packages** in JavaScript projects.
- It helps developers work with external libraries and tools instead of writing everything from scratch.
- Common use cases:
    - installing dependencies
    - running scripts
    - managing project versions
    - handling `package.json`

**Documentation:**
- https://docs.npmjs.com/

---

### Webpack

- **Webpack** is a **module bundler**.
- It takes JavaScript files, CSS, images, and other assets, and bundles them into optimized output files for the browser.
- Common use cases:
    - bundling project files
    - optimizing assets
    - handling loaders and plugins
    - improving performance in production builds

**Documentation:**
- https://webpack.js.org/

---

### Babel

- **Babel** is a **JavaScript compiler / transpiler**.
- It converts modern JavaScript (ES6+) into older JavaScript versions that are supported by more browsers.
- Common use cases:
    - converting modern syntax
    - improving browser compatibility
    - using new JavaScript features safely
    - working with JSX in React

**Documentation:**
- https://babeljs.io/docs/

---

### ESLint

- **ESLint** is a **linting tool** for JavaScript and React code.
- It helps find errors, bad practices, and inconsistent code style before running the application.
- Common use cases:
    - detecting syntax issues
    - enforcing code style rules
    - improving code quality
    - reducing bugs during development

**Documentation:**
- https://eslint.org/docs/latest/

---

### Next.js

- **Next.js** is a **React framework** for building modern web applications.
- It provides routing, server-side rendering, static generation, API routes, and many built-in optimizations.
- Common use cases:
    - building React applications faster
    - file-based routing
    - SSR (Server-Side Rendering)
    - SSG (Static Site Generation)
    - API routes
    - production-ready React apps

**Documentation:**
- https://nextjs.org/docs
